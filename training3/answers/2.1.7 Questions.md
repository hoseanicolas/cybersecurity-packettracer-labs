pix.example.com

5. What are the steps in this attack?
Saat Cafe Customer tersambung ke jaringan nirkabel AP palsu, perangkatnya menerima konfigurasi alamat IP dari server DHCP yang dijalankan pada laptop hacker.
Server DHCP dikonfigurasi untuk mendistribusi alamat laptop hacker sebagai alamat server DNS. Server DNS di laptop hacker dikonfigurasi untuk mengasosiasikan alamat IP server berbahaya 10.6.0.250 dengan nama situs yang dianggap populer "friend.example.com".
Ketika pengguna laptop kafe mencoba mengunjungi situs tersebut, lalu diarahkan ke server berbahaya. Ransomware kemudian diinstal pada laptop pelanggan kafe dan file-file pengguna dienkripsi.
